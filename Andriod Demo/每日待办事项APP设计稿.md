每日待办事项APP设计稿

主要功能：

1. 记录每天待办事项，用户可以新增，并且改变事项状态
2. 设定周期性每日待办事项。比如，这一个月内，每天需要打卡背单词
3. 设定事项的提醒周期。比如，一件学习的任务，会在创建当天的第二天、第三天、第七天、第15天后提示，重新温习

功能架构

1. 整体大模块包含：待办日历、每日待办清单、事项创建、数据统计、个人设置，这5大模块。界面设计上，屏幕下方一共会有5个模块，事项创建按钮居中，相当于左边2个模块按钮，右边2个模块按钮
2. 点击到各个模块后，屏幕上方就会显示相关页面内容，以供下一步操作



### 功能 1：基础每日待办管理

- 新增待办：支持输入待办标题、备注内容
- 事项状态变更：待完成 → 已完成（支持双向切换）
- 编辑 / 删除：支持修改待办内容、删除无效事项

### 功能 2：周期性每日待办（长期打卡）

- 支持设置**固定周期重复**：指定起止时间（如整月每日），自动每日生成待办
- 打卡标记：每日完成周期任务后打卡，未打卡自动标记未完成
- 周期类型：默认「每日重复」，支持自定义重复周期

### 功能 3：智能复习提醒（间隔式提醒）

- 自定义提醒节点：支持设置创建后第 N 天提醒（默认示例：第 2、3、7、15 天）
- 自动触发提醒：到达指定天数，推送温习提醒
- 温习状态：温习完成后更新状态，记录温习次数

我觉得功能1，功能2和功能3可以整合成一个，针对一种事项，可以有2种类型：

类型一：纯完成类事项

新增事项后，会在指定日期当日提醒，默认日期是创建当日，任务完成后即为已完成

类型二：周期类事项

新增事项后，能选择2种，一种是选择一周内的某几天，比如周一至周五

另一种是选择特定的时间点，可以新增多个提醒时间点，比如3天之后，5天之后



我根据可以整合成2种类型事件

创建事项的参数包含：

1. 名称：文本内容，描述事项名字

2. 类型：周期重复/特定时间点

3. 提醒时间：

* 当用户选择周期重复时，提醒时间可选择周一至周日，选择方式可多选
* 当用户选择特定时间点时，提醒时间点默认为一次，默认为创建时的当日，但用户可以点击下方的加号，再选择相应的其他日期。选择日期时，可通过日历视图来点选

4. 事项分类：家庭/学习

这里可以根据用户自定义事项分类名称，每一种类用不同的颜色图标来标注

5. 星标

默认不星标，星标的事项会在每日待办清单中优先显示。用户可点击五角星按钮来编辑是否星标的状态

## 1. 项目概述

### 1.1 项目名称

**DDF** —— day and day follow 跨平台每日待办与智能复习提醒应用

### 1.2 项目目标

开发一款可在 **Windows、iOS、Android** 上运行的网页端应用（PWA），帮助用户高效管理每日待办事项、周期性打卡任务以及间隔式复习提醒。所有数据云端同步，多端实时保持一致。

### 1.3 目标用户

- 需要日常任务管理、习惯打卡的普通用户
- 需要遵循记忆曲线进行知识复习的学生或终身学习者
- 希望在手机、平板、电脑间无缝切换任务的跨设备用户



### 1.4 技术约束

- 开发环境：Windows 机器， Trae
- 最终交付：响应式 Web 应用 + PWA 能力（可安装至主屏幕）
- 数据同步：需要后端服务支持（云数据库 + API）

## 3. 核心功能详细设计

### 3.1 事项类型定义

根据用户需求，将待办事项统一为 **两种类型**，整合了基础待办、周期性打卡、间隔式提醒：

#### 类型一：纯完成类事项（单次）

- **行为**：在指定日期（默认创建当日）出现一次，用户完成后状态变为“已完成”，不可重复。
- **用途**：普通待办、一次性任务。
- **提醒逻辑**：仅在设定的日期当天触发提醒（如有通知权限）。

#### 类型二：周期类事项（重复/多时间点）

- **子类型 A - 按周重复**：选择一周内若干天（如周一、三、五），在每周这些日期自动生成待办。
- **子类型 B - 按特定时间点集合**：用户手动添加多个绝对日期（如 2026-04-20、2026-04-23、2026-04-27），在这些日期分别生成待办。适合复习提醒场景（例如创建后第 2、3、7、15 天）。
- **行为**：每个生成的实例独立记录完成状态。例如一个按周重复事项，周一打卡后不影响周三的未打卡状态。
- **用途**：每日习惯打卡、艾宾浩斯复习计划、固定周期的任务。

- **用途**：每日习惯打卡、艾宾浩斯复习计划、固定周期的任务。



#### 3.2.1 提醒时间控件逻辑

- **当类型 = 纯完成类**：只选择 **一个日期**（默认当天）。用户可修改为未来任意一天。

- **当类型 = 周期类**：需要进一步选择子类型：

  - **子类型 A - 按周重复**：展示周一至周日 7 个开关（多选）。不设结束日期，永久重复（用户可后续手动归档或删除事项）。
  - **子类型 B - 按特定时间点**：展示一个日期列表，初始包含一个日期（默认当天）。点击「+ 添加日期」按钮，弹出日历视图多选或单选更多日期。支持删除已添加的日期。

  ### 3.3 事项分类管理

  - **预设分类**：家庭（🏠 绿色）、学习（📚 蓝色）。用户可在「个人设置」中自定义分类名称和颜色图标。
  - **分类应用**：创建事项时从下拉列表选择。统计页面可按分类筛选。
  - **图标与颜色**：每个分类关联一个 emoji 或字体图标以及一个主题色。

  ### 3.4 星标机制

  - **表现**：在每日待办清单中，所有星标事项排在顶部（按创建时间或自定义排序）。
  - **交互**：事项卡片右侧显示五角星按钮，点击可切换星标状态。支持批量星标（长按进入编辑模式）。

  ### 3.5 每日待办清单界面功能

  - **日期切换**：根据在日期选择器（左右箭头 + 日历快速跳转）中选择的日期，在待办清单界面显示选择日期的待办事项，默认展示当天。
  - **事项列表**：
    - 每个事项显示：名称、分类颜色标记、星标、状态复选框（未完成/已完成）。
    - 纯完成类事项只显示一次。
  - **操作**：
    - 点击复选框切换完成状态（实时同步云端）。
    - 左滑/右滑事项可进行编辑、删除。
    - 点击事项卡片进入详情页（可修改名称、分类、星标、提醒时间等，但修改提醒时间需遵循类型约束）。
  - **空状态**：无待办时显示“暂无任务，点击 + 创建”。

### 3.6 待办日历模块

- **视图**：标准月日历网格。
- **标记方式**：
  - 某天有至少一个未完成事项 → 橙色圆点。
  - 某天所有事项已完成 → 绿色对勾图标。
- **交互**：点击任意日期，自动切换到「每日待办清单」模块并显示该日期的事项。

### 3.7 数据统计模块

- **概览卡片**：
  - 总完成率（所有事项）
  - 本周完成数 / 总事项数
  - 连续打卡天数（基于周期类事项的打卡记录）
- **打卡热力图**：类似 GitHub 贡献图，展示近一年每天完成事项数量。

### 3.8 个人设置模块

- **用户账户**：登录/注册（支持邮箱或手机号），用于数据同步。
- **分类管理**：增删改自定义分类（名称、颜色、图标）。删除分类时提示将该分类下事项移至“未分类”。
- **主题模式**：浅色/深色/跟随系统。
- **数据管理**：导出所有事项为 CSV/JSON，清除全部数据。
- **关于与帮助**：版本号、使用指南。
