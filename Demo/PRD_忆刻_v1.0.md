## 一、产品概述

### 1.1 产品定义

| 维度 | 内容 |
|------|------|
| **产品名称** | 忆刻 |
| **宣传语** | 用碎片时间雕刻记忆 |
| **产品定位** | 科学记忆 + 轻量目标管理，二合一的学习助手 |
| **核心价值** | 基于艾宾浩斯遗忘曲线，帮助用户建立「**计划 → 记忆 → 反馈 → 再计划**」的正循环学习系统 |
| **与竞品差异** | 不是"大而全"的功能堆砌，而是选取记忆卡片的核心流程 + 轻量任务管理，用「Less is more」理念覆盖两类高频场景 |

### 1.2 核心目标

设计一个可提供周期性记忆提醒（符合艾宾浩斯遗忘曲线规律）的学习辅助工具。该产品流程也能复用到具有周期性要求的事件上（如：每月/每周大扫除任务、健身计划等）。

### 1.3 设计初衷

1. **计划与记忆相辅相成**：大多数有知识记忆需求的用户，天然有时间和计划管理的需要。将「做好计划」和「记忆卡片」结合，帮助用户把定期复习、有效输出、建立正反馈形成一个闭环。
2. **填补市场空白**：当前计划类 / 专注类 APP 忽略了"学习"本身建立正反馈的工具。学习的要义在于"记忆"，需要多次复述、深度加工后形成自己的知识库。
3. **拥抱AI趋势**：在后期迭代中引入大语言模型的能力，帮助用户提取知识点核心要义、合理制定目标。

### 3.2 竞品经验教训

**从滴答清单学到**：
- 配色简洁 + 个性化模块设置 = 用户粘性
- ⚠ 功能"大而全"导致学习成本高，忆刻应避免

**从目标地图学到**：
- 聚焦核心场景 + 交互简洁 = 传播力强
- 任务分析内容适合社交媒体分享 → 忆刻应内建分享卡片

**从记乎学到**：
- 团体/班级共享是强留存手段（v2.0参考）
- 第三方卡片付费 = 平台盈利模式（v2.0参考）

**从Anji记忆卡学到**：
- 经典Anki操作习惯用户已教育好，无需重新发明交互
- 复习过程多功能操作（笔记、白板）→ 忆刻MVP先不加，保持简洁
- 统计数据的定期输出对用户很重要

### 5.2 功能详述

#### F-A1/A2：注册 / 登录

| 项目 | 说明 |
|------|------|
| **注册方式** | 手机号 + 验证码（主）；邮箱 + 密码（辅） |
| **登录方式** | 手机号 / 邮箱 + 密码；支持记住登录状态 |
| **最小实现** | MVP允许游客模式（本地存储），注册为可选项。未注册用户数据仅存本地，注册后可同步至云端（v2.0） |
| **异常处理** | 验证码60s重发限制；密码错误3次锁定15分钟；网络异常提示「请检查网络连接」 |
| **隐私合规** | 注册时需勾选《用户协议》和《隐私政策》；明确告知数据使用范围 |

#### F-B1：知识点录入

| 项目 | 说明 |
|------|------|
| **入口** | 首页底部「+」按钮 / 知识面板顶部「+」按钮 |
| **输入方式** | 手动输入（默认）；v2.0扩展：AI解析 / 拍照OCR / 文件导入 |
| **模板选择** | 录入前可选模板，预置字段结构： |
|  | └─ **默认模板**：正面（标题/问题）+ 反面（内容/答案） |
|  | └─ **单词记忆**：正面=单词 + 词性，反面=释义 + 例句 |
|  | └─ **问答题**：正面=题目，反面=答案 + 关键词标签 |
|  | └─ **古诗文**：正面=上句/标题/作者，反面=下句/全文 |
| **分类标签** | 可选，从预设标签选择或自定义输入 |
| **输入限制** | 单面最大5000字；空内容不可保存 |
| **成功反馈** | 保存后Toast「已添加」，自动返回来源页面，新卡片出现在列表顶部 |
| **草稿保存** | 录入中途返回，弹出确认框「是否保存为草稿？」 |

#### F-B2：知识面板（卡片列表）

| 项目 | 说明 |
|------|------|
| **列表展示** | 每张卡片显示：正面内容预览（前50字）+ 分类标签 + 复习轮次 + 上次结果图标（✅/🤔/❌） |
| **排序** | 默认按创建时间倒序；支持按上次复习时间、掌握程度排序 |
| **搜索** | 顶部搜索框，关键词模糊匹配正面/反面内容 |
| **筛选** | 按分类标签筛选 |
| **批量操作** | 长按进入多选模式 → 可批量删除 / 暂停复习 / （v2.0: 批量分享） |
| **空状态** | 无卡片时展示引导图 + 「创建第一张卡片」按钮 |

#### F-B3：知识点详情

| 项目 | 说明 |
|------|------|
| **展示内容** | 正面全文 + 反面全文 + 分类标签 + 创建时间 |
| **复习信息** | 已复习次数、当前记忆阶段、下次复习日期、上次复习结果 |
| **操作** | 编辑 / 删除（二次确认） / 暂停复习 / 分享 |
| **暂停复习** | 暂停后该卡片不再出现在每日复习队列，可随时恢复 |

#### F-C1：首页复习任务

| 项目 | 说明 |
|------|------|
| **首页内容** | 顶部问候语（按时间段）+ 今日待复习卡片数（大数字）+ 进度环 +「开始复习」按钮 |
| **状态展示** | 已完成/未完成状态；全部完成后展示「🎉 今日复习已完成」+ 本次统计摘要 |
| **提醒触达** | 每日定时提醒（默认20:00，可自定义）；到期未完成推送通知 |

#### F-C2：背诵流程（核心交互）

| 项目 | 说明 |
|------|------|
| **入口** | 首页「开始复习」 |
| **步骤** | ① 展示卡片正面 → ② 用户回忆/思考 → ③ 点击翻转或上滑 → ④ 看到反面内容对照 → ⑤ 用户反馈记忆程度 |

**记忆反馈选项（4档）**：

| 选项 | 含义 | 算法行为 |
|------|------|----------|
| ✅ **记住** | 轻松回忆起 | 阶段+1，间隔延长至下一级 |
| 🤔 **模糊** | 部分记得但不确定 | 阶段不变，同会话内再次出现 |
| ❌ **忘记** | 有印象但想不起 | 阶段重置，列入次日复习 |
| 🆕 **完全不记得** | 毫无印象 | 阶段重置，**同会话立即重新学习**（展示完整内容→确认已读→回到队列末尾） |

| 手势支持 | 左滑=忘记 / 右滑=记住 / 上滑=模糊 / 双击=完全不记得 |
| **会话流程** | 队列中所有卡片过完一遍 → 展示本次统计（总数、记住率、模糊率、耗时）→ 返回首页 |
| **中途退出** | 点击「×」→ 弹出「是否结束本次复习？已完成 N/总数」→ 确认后保存进度，下次可继续 |
| **暂停恢复** | 下次进入时，如检测到未完成的会话 → 提示「上次未完成，是否继续？」→ 继续/重新开始 |

#### F-C3：错题本

| 项目 | 说明 |
|------|------|
| **定义** | 聚合展示最近一次复习结果为「忘记」或「完全不记得」的卡片 |
| **入口** | 首页 / 知识面板底部快捷入口 |
| **排序** | 按「忘记次数」倒序（忘记次数越多的越靠前） |
| **操作** | 可单独复习错题；全部重学（进入复习模式） |
| **空状态** | 「👍 没有错题，继续保持！」 |

#### F-C4：复习统计摘要

| 项目 | 说明 |
|------|------|
| **触发** | 每次完成一轮复习后自动弹出 |
| **展示内容** | 本次卡片数 / 记住率 / 模糊率 / 忘记率 / 耗时 |
| **历史对比** | 「比上次 ↑8%」|
| **鼓励文案** | 记住率 ≥80%：「🔥 太棒了！」；50-79%：「继续加油💪」；<50%：「别灰心，重复是记忆之母」 |

#### F-D1：卡片复习轮次

| 项目 | 说明 |
|------|------|
| **展示位置** | 知识面板列表每张卡片下方 / 知识点详情页 |
| **计数规则** | 每完成一次「记住」反馈后轮次+1；模糊/忘记不计 |
| **视觉形式** | 进度条 + 数字（如：已背3轮 / 共需5轮） |

#### F-D2：打卡日历（绿点图）

| 项目 | 说明 |
|------|------|
| **展示形式** | 月视图日历，每日方格内用颜色深浅表示复习量或任务打卡 |
| **颜色规则** | 灰色=未复习 / 浅绿=少量 / 深绿=完成目标 |
| **交互** | 点击某天可查看当日复习卡片数/打卡任务 |
| **激励** | 连续7天 → 徽章提示；连续30天 → 特别庆祝动画 |

#### F-D3：记住率趋势

| 项目 | 说明 |
|------|------|
| **展示** | 折线图，横轴=日期，纵轴=记住率 |
| **时间范围** | 本周 / 本月（切换按钮） |
| **数据源** | 每日复习会话的记住率汇总 |

#### F-E1/E2/E3/E4：任务管理模块

## 八、算法设计

### 8.1 复习间隔计算算法（核心）

**输入参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| current_stage | INTEGER | 当前记忆阶段（0-4） |
| user_result | ENUM | easy / blurry / forgot / lost |
| interval_array | ARRAY | 预设间隔数组，默认 [1, 2, 4, 7, 15] |
| ease_factor | REAL | 易度因子，初始2.5 |

**处理逻辑**：

```
IF user_result == 'easy' THEN
    current_stage = MIN(current_stage + 1, len(interval_array) - 1)
    next_interval = interval_array[current_stage]
    ease_factor = ease_factor + 0.1
    review_count += 1

ELSE IF user_result == 'blurry' THEN
    current_stage = current_stage  (不变)
    next_interval = 0  (同一复习会话内重新出现)

ELSE IF user_result == 'forgot' THEN
    current_stage = 0
    next_interval = interval_array[0]
    ease_factor = MAX(1.3, ease_factor - 0.2)
    forget_count += 1

ELSE IF user_result == 'lost' THEN
    current_stage = 0
    next_interval = 0  (同会话立即重新学习)
    ease_factor = MAX(1.3, ease_factor - 0.3)
    forget_count += 1
END IF

next_review_date = TODAY + next_interval 天
```

**输出**：更新 current_stage, next_review_date, ease_factor, review_count, forget_count

### 8.2 每日复习队列生成算法

```
FUNCTION generate_daily_queue(user_id, today):

    // 1. 收集过期知识点
    expired_cards = SELECT cards WHERE 
        next_review_date <= today 
        AND is_paused = 0
        AND status = 'active'
    ORDER BY next_review_date ASC

    // 2. 获取新知识点（从未复习过）
    new_cards = SELECT cards WHERE 
        card_id NOT IN (SELECT card_id FROM memory_progress)
        AND is_paused = 0
    ORDER BY created_at ASC
    LIMIT daily_new_limit

    // 3. 识别需重学知识点（上次为lost，需要同会话重出）
    relearn_cards = SELECT cards WHERE
        last_result = 'lost'
        AND next_review_date = today

    // 4. 合并并排序
    queue = relearn_cards + expired_cards + new_cards
    // 去重
    queue = DISTINCT(queue)
    // 限制总量
    IF len(queue) > max_review_per_day THEN
        queue = queue[0:max_review_per_day]
    END IF

    RETURN queue
```

### 8.3 新卡片间隔策略

新卡片第一次复习的间隔天数固定为 `interval_array[0]`（默认1天），不参与阶段判断。第二次复习起由算法接管。

```
    │                                        ↓
    │                                  反馈记忆程度
    │                                        │
    │                                        ↓
    └────────── 查看统计 ←── 算法调整间隔 ←──┘
              （正反馈激励）
```

**关键指标（MVP成功标准）**：
- 用户能在3分钟内完成首次「录入→复习」闭环
- 次日留存率目标：≥40%
- 核心操作成功率：≥95%

## 附录 A：术语表

| 术语 | 定义 |
|------|------|
| **卡片** | 一个正面（问题）+ 反面（答案）的知识单元 |
| **复习会话** | 用户从点击「开始复习」到完成所有当日卡片或主动退出的一次完整复习过程 |
| **记忆阶段** | 0-4共5个阶段，决定下次复习间隔天数 |
| **易度因子** | 浮点数，影响间隔计算，初始2.5，记住+0.1，忘记-0.2 |
| **间隔数组** | 默认[1,2,4,7,15]，分别对应阶段0到4的间隔天数 |
| **错题本** | 聚合展示最近结果为「忘记/完全不记得」的卡片 |
| **绿点图** | 类似GitHub贡献图的打卡日历，用颜色深浅表示每日复习量 |
| **记住率** | 一次复习会话中「记住」数 / 总卡片数的百分比 |
