## 一、产品概述

### 1.1 产品定位

| 维度 | 内容 |
|------|------|
| **产品名称** | 下班后8小时（暂定） |
| **一句话定位** | 专为「下班后8小时」设计的时间块 + 待办管理工具 |
| **核心价值** | 用刻度尺时间线的可视化方式，帮助用户规划和追踪下班后的每一段时光，让碎片化时间可量化、可统计 |
| **与通用待办工具的区别** | 不做全天24小时管理，聚焦「下班后的清醒时间」；不是简单清单，而是**先分配时间块，再拆解待办**的二级结构 |

### 1.2 核心场景

- 🏃 **运动健康**：安排跑步、力量训练、瑜伽等，追踪每周运动时长
- 📖 **学习充电**：阅读、网课、技能练习，量化投入时间
- 🎮 **娱乐休闲**：游戏、追剧、社交，了解时间分配是否合理
- 🏠 **生活事务**：家务、购物、陪伴家人

### 1.3 设计理念

```
下班后时间 = 有限资源（~8小时）

任务（大块时间分配）
  ├── 事项1
  ├── 事项2
  └── 事项3

可视化刻度尺 → 一眼看到时间都去哪了
环形图统计   → 知道每周/每月的时间结构
```

## 二、用户与场景分析

### 2.1 目标用户

| 维度 | 描述 |
|------|------|
| **年龄** | 22–40岁职场人 |
| **核心需求** | 下班后想高效利用时间，但缺乏规划和复盘工具 |
| **使用场景** | 下班通勤时规划今晚安排；睡前复盘今日完成情况；周末回顾本周时间结构 |
| **使用频率** | 每日使用（安排+调整），每周回顾统计 |
| **单次使用时长** | 规划 2–3 分钟，查看 < 1 分钟 |

### 2.2 用户痛点

| 序号 | 痛点 | 本工具解法 |
|------|------|------------|
| P1 | 下班后有大量空余时间但不知道怎么安排 | 时间块刻度尺，直观分配时段 |
| P2 | 计划做了但不知道做了多久 | 任务内拆解事项，逐条勾选 |
| P3 | 一周过去了不知道时间花在哪 | 环形图/柱形图多维度统计 |
| P4 | 通用待办工具太复杂、不符合下班场景 | 聚焦8小时 + 二级结构，极简操作 |

## 四、功能详述

### 4.1 今日安排

#### 4.1.1 时间刻度尺（左半侧）

| 项目 | 说明 |
|------|------|
| **展示范围** | 默认 18:00 – 次日 02:00（可滚动延伸） |
| **显示内容** | 纵轴时间刻度（每30分钟一格）+ 已分配任务块（彩色填充） |
| **任务块** | 每个任务占据对应的时段时间区间，不同任务用不同颜色填充 |
| **空闲时间** | 灰色/浅色背景显示，暗示「可用」 |
| **重叠处理** | 同一时间段如有多任务，用半透明叠加 + 边缘颜色标识，点击可切换 |
| **交互** | 点击任务块 → 右侧高亮该任务的待办列表；长按可拖动调整时间段 |

**颜色预设**（可自定义）：

| 颜色 | 推荐场景 |
|------|----------|
| 🟢 绿色 | 运动健康 |
| 🔵 蓝色 | 学习充电 |
| 🟡 黄色 | 生活事务 |
| 🟣 紫色 | 娱乐休闲 |
| 🔴 红色/橙色 | 重要紧急 |

#### 4.1.2 任务待办列表（右半侧）

| 项目 | 说明 |
|------|------|
| **触发** | 点击左侧任务块 / 默认显示当前时段的任务 |
| **展示内容** | 任务标题 + 颜色标签 + 时间段（如 21:00–22:00）+ 逐条待办事项 |
| **待办条目** | 序号 + 事项内容 + 优先级标识（高/中/低） + 勾选框 |
| **勾选反馈** | 勾选后事项文字加删除线，同步刻度尺上任务块显示完成进度百分比 |
| **全部完成** | 任务块变为实心填充 + 外圈高亮，右侧显示「✅ 全部完成」 |

**示例布局**：

```
┌──────────────┬─────────────────────┐
│  刻度尺区域   │  任务待办列表        │
│              │                     │
│  22:00 ─     │  🟢 锻炼身体        │
│        │     │  21:00–22:00        │
│  21:30 │ 🟢  │                     │
│        │ 锻  │  □ 1. 跑步30分钟     │
│  21:00 │ 炼  │  □ 2. 无氧20分钟     │
│        │     │  □ 3. 拉伸10分钟     │
│  20:30 ─     │                     │
│        │ 🔵  │  🔵 阅读充电        │
│  20:00 │ 阅  │  20:00–20:30        │
│        │ 读  │  □ 1. 《xxx》第3章   │
│  19:30 ─     │  □ 2. 笔记整理       │
│        │     │                     │
│  19:00 │     │                     │
│        │     │                     │
│  18:30 ─     │                     │
└──────────────┴─────────────────────┘
```

#### 4.1.3 时间冲突与重叠

| 场景 | 处理方式 |
|------|----------|
| 新建任务时间段与其他任务重叠 | 弹出提示「该时段已有『XXX』任务，是否覆盖？」→ 覆盖/调整 |
| 任务超时（实际执行超出分配时间） | 手动拖拽边界延长，刻度尺自动重排后续任务 |
| 空闲时间单击 | 弹出「添加任务」入口，预设当前点击的时间段 |

### 4.2 新增事项

#### 4.2.1 入口

- 底部导航栏中间「+」按钮
- 刻度尺空闲区域单击 → 快捷创建任务

#### 4.2.2 新增任务（时间块）

| 字段 | 说明 | 必填 |
|------|------|------|
| **任务标题** | 如「锻炼身体」「学习英语」 | ✅ |
| **时间段** | 开始时间 + 结束时间（滚轮选择器） | ✅ |
| **颜色** | 从预设8色中选择，或自定义色值 | ✅ |
| **重复** | 仅今天 / 每天 / 每周X / 自定义（MVP阶段为「仅今天」） | — |
| **备注** | 自由文本 | — |

**交互流程**：

```
点击「+」→ 选择「新建任务」
    → 输入标题
    → 选择开始/结束时间（滚轮）
    → 选择颜色
    → 保存 → 刻度尺出现新任务块 → 右侧自动进入事项编辑
```

#### 4.2.3 新增事项（任务下的待办）

| 字段 | 说明 | 必填 |
|------|------|------|
| **所属任务** | 选择一个已有任务（下拉列表） | ✅ |
| **事项内容** | 如「跑步30分钟」 | ✅ |
| **优先级** | 高 / 中 / 低（默认：中） | — |
| **颜色** | 自动继承所属任务颜色，不可单独修改 | — |

**交互流程**：

```
点击「+」→ 选择「新建事项」
    → 选择所属任务（从今日任务列表选）
    → 输入事项内容
    → 选择优先级
    → 保存 → 事项出现在该任务的待办列表中
```

#### 4.2.4 编辑与删除

| 操作 | 方式 |
|------|------|
| 编辑任务 | 长按任务块 →「编辑」→ 修改字段 → 保存 |
| 删除任务 | 长按任务块 →「删除」→ 二次确认「删除后该任务下所有事项也会删除」 |
| 编辑事项 | 右滑事项 →「编辑」按钮 |
| 删除事项 | 左滑事项 →「删除」按钮（无二次确认，即时删除） |

### 4.3 时间统计

#### 4.3.1 环形图（时间占比）

| 项目 | 说明 |
|------|------|
| **数据来源** | 各任务的分配时长（非实际打卡时长） |
| **展示方式** | 环形图，不同扇区 = 不同任务（颜色与任务颜色一致） |
| **切换维度** | 本周 / 本月（顶部Tab切换） |
| **中心显示** | 总分配小时数 |
| **扇区交互** | 点击扇区 → 弹出该任务的耗时明细（多少天、占比、总事项数） |

**示例**：

```
        ┌──────────────────┐
        │    📊 时间统计    │
        │   [本周] [本月]   │
        │                  │
        │    ◯ 环形图      │
        │   🟢 运动 35%   │
        │   🔵 学习 28%   │
        │   🟣 娱乐 22%   │
        │   🟡 生活 15%   │
        │                  │
        │  总：32小时      │
        └──────────────────┘
```

#### 4.3.2 柱形图（任务耗时对比）

| 项目 | 说明 |
|------|------|
| **展示** | 柱形图，横轴 = 任务名称，纵轴 = 总耗时（小时） |
| **场景** | 快速对比各任务的投入情况 |
| **切换** | 本周 / 本月 |
| **排序** | 按耗时降序排列 |

#### 4.3.3 折线图（事项数量趋势）

| 项目 | 说明 |
|------|------|
| **展示** | 折线图，横轴 = 日期，纵轴 = 当日事项完成数 |
| **多线** | 可按任务分类展示多条折线（如运动事项 vs 学习事项） |
| **时间范围** | 本周 / 本月 |

### 4.4 个人设置

#### 4.4.1 入口与交互

- **位置**：主界面右上角齿轮图标 ⚙
- **交互**：点击后从右侧滑入设置面板（抽屉式，覆盖当前页面）

| 分类 | 设置项 | 说明 |

```
│ phone    │       │ user_id  FK  │       │ task_id  FK  │
│ nickname │       │ title        │       │ content      │
│ theme    │       │ color        │       │ priority     │
│ ...      │       │ start_time   │       │ is_done      │
└──────────┘       │ end_time     │       │ sort_order   │
                   │ date         │       │ created_at   │
                   │ repeat_rule  │       └──────────────┘
                   │ created_at   │
                   └──────────────┘
```

### 6.2 表结构

#### user（用户表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| user_id | TEXT | PK | UUID |
| phone | TEXT | UNIQUE | 手机号 |
| email | TEXT | UNIQUE | 邮箱 |
| nickname | TEXT | | 昵称 |
| theme | TEXT | DEFAULT 'light' | light / dark |
| reminder_plan | TEXT | | 规划提醒时间（HH:MM） |
| reminder_task | INTEGER | DEFAULT 1 | 任务开始提醒 0/1 |
| created_at | INTEGER | | Unix时间戳 |

#### task（任务表，时间块）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| task_id | TEXT | PK | UUID |
| user_id | TEXT | FK → user | 所属用户 |
| title | TEXT | NOT NULL | 任务标题 |
| color | TEXT | NOT NULL | 色值（如 #4CAF50） |
| start_time | TEXT | NOT NULL | 开始时间（HH:MM） |
| end_time | TEXT | NOT NULL | 结束时间（HH:MM） |
| date | TEXT | NOT NULL | 日期（YYYY-MM-DD） |
| repeat_rule | TEXT | DEFAULT 'none' | none / daily / weekly:N |
| note | TEXT | | 备注 |
| created_at | INTEGER | | Unix时间戳 |

#### todo_item（待办事项表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| item_id | TEXT | PK | UUID |
| task_id | TEXT | FK → task | 所属任务 |
| content | TEXT | NOT NULL | 事项内容 |
| priority | TEXT | DEFAULT 'medium' | high / medium / low |
| is_done | INTEGER | DEFAULT 0 | 是否完成 |
| sort_order | INTEGER | DEFAULT 0 | 排序序号 |
| created_at | INTEGER | | Unix时间戳 |
| done_at | INTEGER | | 完成时间戳 |

### 6.3 本地存储策略

- **技术选型**：SQLite（HarmonyOS 关系型数据库）
- **存储位置**：本地沙箱，不上传服务器（MVP阶段）
- **备份**：支持导出 JSON 文件
